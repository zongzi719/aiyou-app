# Luna 项目常用命令（给非开发同学用）

所有命令都在 **Mac 终端** 里执行。先打开「终端」，输入 `cd ` 后把 **luna-main 项目文件夹** 拖进终端窗口，回车，就到了项目根目录。

---

## 一、API 地址怎么配（默认直连线上网关）

项目已默认使用线上 AIYOU 网关：**`http://47.242.248.240:2026`**。

- **移动端（模拟器 / 真机）**：根目录 `.env` 里配置 `EXPO_PUBLIC_DEV_API_BASE_URL`（与示例一致即可），改完后需重新执行一次 **`npx expo run:ios`**（或完整重启 Metro + App）才会打进包里。
- **开发者工具里的「环境配置」**：仅 **`npx expo start --web`** 后在浏览器打开（例如终端里按 `w`，再进 **`/dev/api-settings`**）。原生 App 内 `/dev` 为 Web 专用入口，模拟器深链不会打开该页。

在 Web 环境配置页可点 **「使用线上网关」** 一键填回 `http://47.242.248.240:2026`，保存后 Dev 工具里的直连请求会走该地址。

---

## 二、日常跑 App（模拟器）

```bash
cd /你的路径/luna-main
npm install
npx expo run:ios
```

第一次会久一点。以后改 **只有 JS** 时，可只开 Metro：

```bash
npm run start
```

然后在模拟器里点开 Luna（需已用上面命令装过开发版）。

## 三、登录页路径（方便对照）

- 登录：**`/screens/login`**
- 用命令打开示例：

```bash
xcrun simctl openurl booted "luna://screens/login"
```

## 四、（可选）网络异常时的本机转发代理

**默认不需要。** 仅当模拟器直连网关失败、而 Mac 上 `curl` 能通时，可另开终端运行 `npm run dev:api-proxy`，把 API 地址改为 **Mac 局域网 IP + 代理端口**（勿用 `127.0.0.1`，模拟器里指向的是模拟器自身）。详见脚本 `scripts/dev-api-proxy.mjs` 内说明。

---

## 五、在 Mac 上测服务器能不能连（给技术支持看结果）

```bash
curl -sS -o /dev/null -w "%{http_code}\n" -X POST "http://47.242.248.240:2026/api/auth/user-login" -H "Content-Type: application/json" -d '{"username":"test","password":"test"}'
```

- 若最后一行是 **401**：说明 Mac 能连上服务器（密码错是正常的）。
- 若一直卡住或报错：说明当前网络到服务器不通，要先换网络或找管理员放行端口。

## 六、其它 npm 脚本（可选）

| 命令 | 作用 |
|------|------|
| `npm run start` | 启动 Metro（开发客户端） |
| `npm run ios` | 同 `expo run:ios`，编译并跑 iOS |
| `npm run android` | 编译并跑 Android |
| `npm run dev:api-proxy` | （可选）本机 API 转发，见第四节 |
| `npm run lint` | 代码检查（开发用） |

---

## 七、仍打不开页面时

1. 确认 **模拟器已启动**，且 **Luna 已安装**（至少成功跑过一次 `npx expo run:ios`）。
2. 确认项目 `app.json` 里 **scheme** 是 `luna`（与上面 `luna://` 一致）。
3. 把终端里 **完整报错** 截图发给技术支持。

## 八、Git 分支开发流程（功能分支 → PR 合并）

### 开始新功能
```bash
git checkout main                    # 回到 main
git pull origin main                 # 拉取最新代码
git checkout -b feat/功能名称         # 新建并切换功能分支
```

### 开发中提交
```bash
git add .
git commit -m "feat: 描述本次改动"
```

### 推送到 GitHub
```bash
git push -u origin feat/功能名称     # 首次推送用 -u，之后直接 git push
```

### 合并后清理
```bash
git checkout main
git pull origin main                 # 同步合并结果
git branch -d feat/功能名称          # 删除本地分支
```

### 分支命名规范
| 类型 | 格式 | 示例 |
|------|------|------|
| 新功能 | `feat/xxx` | `feat/knowledge-base` |
| Bug 修复 | `fix/xxx` | `fix/upload-crash` |
| UI 调整 | `ui/xxx` | `ui/drawer-style` |
