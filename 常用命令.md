# Luna 项目常用命令（给非开发同学用）

所有命令都在 **Mac 终端** 里执行。先打开「终端」，输入 `cd ` 后把 **luna-main 项目文件夹** 拖进终端窗口，回车，就到了项目根目录。

---

## 一、怎么打开「环境配置」（改 API 地址）

环境配置页在 App 里的路径是：**`/dev/api-settings`**（和网页路径一样）。

### 方法 A：用 Mac 命令直接跳进环境配置（推荐）

1. 先像平时一样 **`npm run start`** 或 **`npx expo run:ios`**，让模拟器和 Metro 都跑起来。
2. 再开一个新的终端窗口，执行（整行复制）：

```bash
xcrun simctl openurl booted "luna://dev/api-settings"
```

若没反应，可试：

```bash
xcrun simctl openurl booted "luna:///dev/api-settings"
```

3. 模拟器里的 Luna 应会打开 **环境配置** 页。在这里改 **API 基础地址**，点 **保存**。

### 方法 B：先打开开发者工具首页，再点进去

在终端执行：

```bash
xcrun simctl openurl booted "luna://dev"
```

进入 **开发者工具 /dev** 后，点列表里的 **「环境配置」**。

---

## 二、日常跑 App（模拟器）

```bash
cd /你的路径/luna-main
npm install
npx expo run:ios
```

第一次会久一点。以后改 **只有 JS** 时，可只开 Metro：

```bash
npm run start
```

然后在模拟器里点开 Luna（需已用上面命令装过开发版）。

---

## 三、登录页路径（方便对照）

- 登录：**`/screens/login`**
- 用命令打开示例：

```bash
xcrun simctl openurl booted "luna://screens/login"
```

---

## 四、网络不通时：本机代理（API 走 Mac再转发）

当模拟器直连 `http://47.x.x.x:2026` 失败、但 Mac 上 `curl` 能通时，在项目根目录 **另开一个终端** 保持运行：

```bash
npm run dev:api-proxy
```

看到提示监听 `http://127.0.0.1:13026` 后，去 **环境配置** 把 API 基础地址改成：

`http://127.0.0.1:13026`

保存后再试登录。

---

## 五、在 Mac 上测服务器能不能连（给技术支持看结果）

```bash
curl -sS -o /dev/null -w "%{http_code}\n" -X POST "http://47.242.248.240:2026/api/auth/user-login" -H "Content-Type: application/json" -d '{"username":"test","password":"test"}'
```

- 若最后一行是 **401**：说明 Mac 能连上服务器（密码错是正常的）。
- 若一直卡住或报错：说明当前网络到服务器不通，要先换网络或找管理员放行端口。

## 六、其它 npm 脚本（可选）

| 命令 | 作用 |
|------|------|
| `npm run start` | 启动 Metro（开发客户端） |
| `npm run ios` | 同 `expo run:ios`，编译并跑 iOS |
| `npm run android` | 编译并跑 Android |
| `npm run dev:api-proxy` | 本机 API 转发代理（见第四节） |
| `npm run lint` | 代码检查（开发用） |

---

## 七、仍打不开页面时

1. 确认 **模拟器已启动**，且 **Luna 已安装**（至少成功跑过一次 `npx expo run:ios`）。
2. 确认项目 `app.json` 里 **scheme** 是 `luna`（与上面 `luna://` 一致）。
3. 把终端里 **完整报错** 截图发给技术支持。
